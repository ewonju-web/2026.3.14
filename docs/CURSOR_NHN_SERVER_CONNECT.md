# Cursor에서 NHN 서버 연결 후 파악·다운로드

기존 서버(211.110.140.201) 연결하듯이 **NHN 서버도 Cursor에 등록**하고, SSH로 접속해서 웹 루트를 탐색·다운로드하는 방법입니다.

---

## 1. NHN 서버 정보 확인

NHN 호스팅/클라우드 콘솔에서 아래를 확인합니다.

| 항목 | 예시 | 비고 |
|------|------|------|
| **호스트(IP 또는 도메인)** | `123.45.67.89` 또는 `nhn-server-xxx.nhncloud.com` | |
| **SSH 포트** | `22` (기본) 또는 `10022` 등 | |
| **접속 사용자** | `ubuntu`, `root`, `nhn` 등 | |
| **인증 방식** | 비밀번호 또는 **SSH 키** | 키 사용 시 키 파일 경로 확인 |

---

## 2. SSH 설정 파일에 NHN 서버 추가

PC에서 **SSH config**를 수정해 NHN 서버를 하나의 호스트 이름으로 등록합니다.

### 2.1 설정 파일 위치

- **Windows**: `C:\Users\본인사용자명\.ssh\config`
- **macOS / Linux**: `~/.ssh/config`

파일이 없으면 새로 만들고, `chmod 600 config` (리눅스/맥) 적용합니다.

### 2.2 NHN 서버 블록 추가

아래에서 `NHN서버IP`, `사용자`, `키경로` 등을 NHN에서 안내한 값으로 바꿉니다.

```sshconfig
# 기존 서버 (예: 클론 서버)
Host excavator
    HostName 211.110.140.201
    User root
    Port 22
    # IdentityFile ~/.ssh/your_key

# NHN 서버 (direct-nara 백업/파악용)
Host nhn
    HostName NHN서버IP
    User NHN접속사용자
    Port 22
    # 비밀번호 로그인: 아래 두 줄 생략
    # 키 로그인:
    IdentityFile ~/.ssh/nhn_key
```

- **비밀번호만 쓰는 경우**: `IdentityFile` 줄을 넣지 않으면, 접속 시 비밀번호를 묻습니다.
- **Port**가 22가 아니면 (예: 10022) `Port 10022` 로 지정합니다.

## 3. Cursor에서 NHN 서버 연결

1. **Cursor** 실행
2. **F1** 또는 **Ctrl+Shift+P** (맥: Cmd+Shift+P) → **“Remote-SSH: Connect to Host…”** 입력 후 선택
3. 목록에서 **`nhn`** (또는 위에서 쓴 `Host` 이름) 선택
4. 새 Cursor 창이 열리면 **터미널**과 **파일 탐색**이 NHN 서버 기준으로 동작합니다
5. **파일 → 폴더 열기** → NHN 서버에서 **웹 루트 경로** 입력  
   (예: `/home/nhn/public_html`, `/var/www/html`, NHN 안내 경로 등)

이제 **기존 서버 연결하듯이** NHN 서버 안을 Cursor에서 탐색할 수 있습니다.

---

## 4. 서버 구조 파악 후 다운로드

### 4.1 파악

- 왼쪽 **탐색기**에서 웹 루트 폴더를 열어 보면:
  - PHP/HTML 소스, `uploads`/`media`, `config` 등 구조를 그대로 확인할 수 있습니다.
- **터미널** (Cursor 하단)에서:
  - `ls -la`, `find . -name "*.php"` 등으로 경로·파일을 확인할 수 있습니다.
- **DB 설정**이 있다면 `config.php`, `.env` 등에서 DB 종류·접속 정보를 확인할 수 있습니다.

### 4.2 다운로드(백업)

**방법 A: Cursor에서 직접 복사**

- 탐색기에서 **폴더/파일 우클릭 → Download** (또는 드래그로 로컬로 복사)
- 웹 루트 전체를 선택해 다운로드하면 PC 로컬에 백업됩니다.

**방법 B: 터미널에서 압축 후 다운로드**

1. Cursor에서 NHN 서버에 연결된 **터미널** 열기
2. 웹 루트 상위로 이동 후 압축:

```bash
cd /경로/웹루트상위
tar czvf ~/webroot_backup_$(date +%Y%m%d).tar.gz 웹루트폴더이름
```

3. 탐색기에서 `~/webroot_backup_*.tar.gz` 파일 **우클릭 → Download**  
   → PC로 한 번에 받을 수 있습니다.

**방법 C: 로컬 PC 터미널에서 SCP**

- Cursor 없이, PC에서:

```bash
scp -r nhn:/경로/웹루트폴더 ./local_backup/
```

## 5. 받은 백업 파일 전달 방법

NHN에서 받은 백업(웹 루트·DB 덤프)을 **어디로 옮길지**에 따라 방법이 나뉩니다.

### 방법 1: PC에만 두고, 나중에 필요할 때 업로드

- 백업은 **PC 로컬**에만 둡니다.
- 나중에 **현재 서버(211.110.140.201)** 나 **NHN 새 서버**에 올릴 때, PC에서 업로드합니다.

**PC → 현재 서버(211.110.140.201)로 보낼 때 (PC 터미널에서):**

```bash
scp -r ./local_backup_folder 사용자@211.110.140.201:/tmp/
# 또는 압축 파일 하나만
scp ./webroot_backup_20260307.tar.gz 사용자@211.110.140.201:/tmp/
```

- `사용자`는 211.110.140.201 SSH 접속 계정 (예: root).  
- 받는 쪽 경로는 `/tmp/` 대신 `/srv/excavator/` 등 원하는 곳으로 바꾸면 됩니다.

**PC → NHN 서버로 보낼 때:**  
위와 같이 `사용자@NHN서버IP:/경로` 로 `scp` 하면 됩니다.

---

### 방법 2: NHN 서버 → 현재 서버로 직접 복사 (PC 거치지 않음)

Cursor로 **NHN 서버에 연결된 터미널**에서, 현재 서버(211.110.140.201)로 바로 보낼 수 있습니다.

**NHN 서버 터미널에서:**

```bash
# NHN 서버에서 211.110.140.201로 전송 (서버→서버)
scp -r /경로/웹루트백업 사용자@211.110.140.201:/tmp/
# 또는 압축 후 한 번에
tar czvf - 웹루트폴더 | ssh 사용자@211.110.140.201 "cat > /tmp/webroot_backup.tar.gz"
```

- 211.110.140.201 쪽에서 **NHN 서버 SSH 접속을 허용**해야 하고,  
  NHN 서버에서 211.110.140.201로 나갈 수 있는 네트워크/방화벽이 열려 있어야 합니다.  
  (회사/호스팅 정책에 따라 막혀 있을 수 있음.)

---

### 방법 3: Cursor로 현재 서버 열고, 거기로 업로드

- **Cursor**에서 **Remote-SSH → 211.110.140.201(기존 서버)** 로 연결한 창을 엽니다.
- 그 창에서는 **파일 탐색기**가 211.110.140.201 기준입니다.  
  여기로 “전달”하려면 **PC에 있는 파일을 이 서버에 올려야** 하므로, **현재 서버 터미널**에서 PC를 거꾸로 받는 건 Cursor만으로는 어렵습니다.
- 대신 **PC에서** 위 방법 1처럼 `scp` 로 211.110.140.201에 업로드한 뒤,  
  Cursor(211.110.140.201 연결)에서 `/tmp/` 등 올려둔 경로를 열어서 확인·사용하면 됩니다.

---

## 6. 요약

| 단계 | 작업 |
|------|------|
| 1 | NHN 콘솔에서 서버 IP, 사용자, 포트, 키/비밀번호 확인 |
| 2 | PC `~/.ssh/config`에 `Host nhn` 추가 |
| 3 | Cursor → **Remote-SSH: Connect to Host** → `nhn` 선택 |
| 4 | **폴더 열기**로 웹 루트 열어서 구조 파악 |
| 5 | 탐색기에서 **Download** 또는 터미널에서 **압축 후 다운로드** |
| 6 | **전달**: PC에 둔 뒤 필요할 때 `scp`로 211.110.140.201 또는 NHN 서버로 업로드 (또는 NHN→211 서버 직접 scp) |

이렇게 하면 Cursor에 NHN 정보를 넣고, 기존 서버 연결하듯이 NHN 서버에 연결한 뒤 파악·다운로드하고, 받은 백업을 원하는 서버로 전달할 수 있습니다.
