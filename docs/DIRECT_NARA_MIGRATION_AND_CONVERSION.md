# direct-nara.co.kr → 굴삭기나라 DB 이관 및 정식 회원 전환

direct-nara는 **전화번호·성함(닉네임)·비밀번호**로 회원가입했고, 여기서 만든 사이트(굴삭기나라)는 Django **User(username, email, password)** + **Profile(전화, 상호 등)** 구조입니다.  
이 문서는 **기존 회원·매물을 새 DB에 적용하는 방법**과 **이관 회원의 정식 회원 전환** 절차를 정리한 것입니다.

---

## 0. 기존 회원(전화번호+이름+비밀번호) 최대한 간편하게 가입시키는 방법

예전처럼 **전화번호와 이름, 비밀번호**로만 쓰던 회원들을 새 사이트에서 **최소한의 단계**로 쓸 수 있게 하는 흐름입니다.

### 전체 흐름 요약

| 순서 | 누가 | 무엇을 |
|------|------|--------|
| 1 | 운영자 | DB 이관 (`import_direct_nara`) → 기존 회원이 `legacy_회원번호` 계정으로 새 DB에 들어감 |
| 2 | 운영자 | 각 이관 회원에게 **첫 로그인용** 안내 (아래 2가지 중 선택) |
| 3 | 기존 회원 | **한 번만** 로그인 → 마이페이지에서 **「정식 회원 전환」** → 새 아이디·이메일·비밀번호 설정 |
| 4 | 이후 | 일반 회원처럼 **새 아이디 + 새 비밀번호**로 로그인 |

새 사이트는 “아이디(username) + 비밀번호”로 로그인하므로, **처음 한 번**은 “이관용 아이디(legacy_xxx) + 임시 비밀번호”로 들어와서, **정식 회원 전환**에서 본인이 쓸 **새 아이디·이메일·비밀번호**를 정하면 됩니다.

### 기존 회원에게 안내할 수 있는 두 가지 (가장 간편한 것부터)

**방법 A) 관리자 임시 비밀번호 (가장 간단)**

1. 관리자 페이지 → **회원(User)** 또는 **사용자 프로필(Profile)** 목록에서 해당 회원 선택.
2. **비밀번호 변경**에서 임시 비밀번호 입력 후 저장 (예: `temp1234`).
3. 회원에게 **문자/전화**로 아래만 안내하면 됩니다.
   - *“굴삭기나라 새 사이트 주소: https://도메인”*
   - *“로그인 아이디: legacy_회원번호”* (관리자 화면에서 확인)
   - *“임시 비밀번호: (설정한 값)”*
   - *“로그인 후 **마이페이지 → 정식 회원 전환**에서 **새 아이디·이메일·비밀번호** 한 번만 설정해 주세요.”*
4. 회원이 로그인 → 마이페이지 → **「정식 회원 전환」** 클릭 → 새 아이디·이메일·비밀번호 입력 → 저장.
5. 이후부터는 **새 아이디 + 새 비밀번호**로 로그인하면 됩니다.

→ **가입 절차는 “로그인 1번 + 정식 회원 전환 1번”** 이라서, 기존 회원 입장에서는 가장 간편합니다.

**방법 B) 전화번호로 찾기 + 비밀번호 설정 (추가 구현 시)**

- “비밀번호 찾기”처럼 **전화번호 입력** → 해당하는 이관 회원(legacy)이 있으면, **이메일 또는 문자**로 “비밀번호 설정 링크” 발송.
- 회원이 링크에서 **비밀번호만 설정** → 설정 완료 시 자동 로그인(또는 로그인 페이지로 이동).
- 로그인 후 마이페이지에서 **정식 회원 전환**으로 새 아이디·이메일·비밀번호 설정.

→ 이메일/문자 발송이 가능할 때, “아이디를 모르는 회원”도 전화번호만으로 진입할 수 있어 더 편합니다.

### 한 줄 요약

- **이관**으로 기존 회원을 `legacy_회원번호` 계정으로 새 DB에 넣고,
- **임시 비밀번호**를 알려주어 **한 번 로그인**하게 한 뒤,
- **정식 회원 전환** 한 번으로 **새 아이디·이메일·비밀번호**를 정하게 하면,  
  전화번호+이름+비밀번호로 쓰던 기존 회원을 **최대한 간편하게** 새 사이트에 가입(전환)시킬 수 있습니다.

## 1. DB 이관 방법 (의견)

### 1.1 현재 구조 정리

| 구분 | direct-nara (기존) | 굴삭기나라 (현재) |
|------|---------------------|-------------------|
| 회원 | 전화번호, 성함/닉네임, 비밀번호 | User: username, email, first_name, password / Profile: phone, company_name |
| 매물 | tb_pro (uid, 작성자 연락처 등) | Equipment (author=User FK) |

- **이관 스크립트**: `equipment/management/commands/import_direct_nara.py`  
  - direct-nara MySQL DB(`sw_member`, `tb_pro`) → User, Profile, Equipment 생성
  - 회원은 `username=legacy_{mb_num}`, 비밀번호는 `set_unusable_password()` 로 두고 이관 (로그인 불가 → 정식 전환 시 비밀번호 설정)
  - 매물은 기존 작성자 전화번호로 회원 매핑 후 `author` 연결
- **주의**: 과거 마이그레이션에서 `legacy_member_id`, `legacy_listing_id` 필드가 제거된 상태라면, **재이관·중복 방지**를 위해 이 필드를 다시 넣고 `import_direct_nara` 를 쓰는 것을 권장합니다.

### 1.2 권장 이관 순서

1. **DB 스냅샷**  
   - direct-nara DB를 dump 해서 **이관 기준 시점** 고정 (테스트/본이관 동일 스냅 사용).

2. **설정**  
   - `config/settings.py` 의 `DATABASES` 에 direct-nara용 DB 추가 (예: `direct_nara`).
   - 예시:
     ```python
     DATABASES = {
         "default": { ... },  # 현재 사이트 DB
         "direct_nara": {
             "ENGINE": "django.db.backends.mysql",
             "NAME": "direct_nara_db",
             "USER": "...",
             "PASSWORD": "...",
             "HOST": "...",
         },
     }
     ```
   - `pip install mysqlclient` 또는 `pymysql` 사용.

3. **Legacy 필드 복구 (필요 시)**  
   - Profile 에 `legacy_member_id` (IntegerField, null=True), Equipment 에 `legacy_listing_id` (IntegerField, null=True) 가 없으면 마이그레이션으로 추가.  
   - 있으면 `import_direct_nara` 가 그대로 동작해 **재실행 시 중복 생성 방지** 가능.

4. **회원 이관**  
   ```bash
   python manage.py import_direct_nara --dry-run   # 건수 확인
   python manage.py import_direct_nara             # 실제 이관
   ```
   - 회원: `username=legacy_{mb_num}`, 전화/성함은 Profile·User.first_name 에 저장.
   - 비밀번호는 `set_unusable_password()` 로 두고, **정식 회원 전환**에서 새 비밀번호 설정하도록 하는 방식을 권장.

5. **매물 이관**  
   - 위 명령 한 번에 회원+매물 같이 이관됨.  
   - 매물만 다시 돌리려면: `python manage.py import_direct_nara --listings-only`

6. **이미지 이관**  
   - `import_direct_nara_images` 등 기존 이미지 이관 명령이 있다면, 매물 이관 후 같은 규칙(예: legacy_listing_id 매칭)으로 실행.

### 1.3 비밀번호 처리

- direct-nara 비밀번호 해시 방식이 Django와 **같으면** (예: PBKDF2) 이관 시 해시를 그대로 옮겨 넣을 수 있음 (구현 시 주의).
- **다르거나 불확실하면**:  
  - 이관 시 `set_unusable_password()` 로 두고,  
  - **정식 회원 전환** 또는 **비밀번호 찾기(이메일)** 로 최초 비밀번호 설정하게 하는 것이 안전합니다.

## 2. 정식 회원 전환 방법

**정식 회원 전환**이란, `legacy_123` 같은 **이관 전용 아이디**를 쓰는 회원이  
**일반 회원처럼 로그인·서비스 이용**할 수 있도록 **아이디·이메일·비밀번호**를 정리하는 절차입니다.

### 2.1 전제

- 이관 시 비밀번호를 `set_unusable_password()` 로 두었다면, **최초 로그인은 불가**합니다.
- 따라서 다음 둘 중 하나가 필요합니다.
  - **A) 관리자가 임시 비밀번호 부여 (권장)**  
    - Django 관리자 → 사용자 → 해당 User 선택 → “비밀번호 변경”에서 임시 비밀번호 입력 후 저장.  
    - 회원에게 전화/문자로 `아이디: legacy_123`, `임시 비밀번호: xxx` 전달.  
    - 회원이 **해당 아이디(legacy_123) + 임시 비밀번호**로 로그인 → 마이페이지에서 **「정식 회원 전환」** 클릭 → 새 아이디·이메일·비밀번호 설정.
  - **B) 전화번호·이메일로 본인 확인 후 비밀번호 설정**  
    - “비밀번호 찾기”와 유사하게, 전화번호 + 이메일로 본인 확인 후 한 번 쓰는 링크에서 비밀번호 설정.  
    - 구현 시: 전화번호로 User/Profile 찾기 → 이메일 발송 → 링크에서 비밀번호 설정 + (선택) 정식 전환 유도.

### 2.2 정식 전환 화면 (구현됨)

- **URL**: `/account/convert/` (이름: `legacy_convert`)
- **대상**: 로그인한 회원 중 `username` 이 `legacy_` 로 시작하는 경우.  
  마이페이지에서 **「정식 회원 전환 (아이디·이메일·비밀번호 설정)」** 버튼이 보입니다.
- **입력 항목**  
  - **새 로그인 아이디**: `legacy_` 로 시작하지 않는, 사이트에서 쓸 아이디 (중복 불가).  
  - **이메일**: 비밀번호 찾기·알림용.  
  - **새 비밀번호** + 확인 (8자 이상).
- **처리**  
  - `User.username`, `User.email` 갱신, `User.set_password(새 비밀번호)` 후 저장.  
  - 세션 유지(`update_session_auth_hash`)로 즉시 로그아웃되지 않음.  
  - 전환 후에는 아이디가 바뀌므로 정식 전환 배너는 더 이상 노출되지 않음.

### 2.3 전환 후

- 해당 회원은 **일반 회원과 동일**하게 로그인·비밀번호 찾기·마이페이지 이용 가능.
- 기존에 매물만 이관된 경우에도 `author` 가 이미 User 에 연결되어 있으므로, **추가 DB 작업 없이** 그대로 “내 매물”로 보이게 할 수 있습니다.

## 3. 요약

| 단계 | 내용 |
|------|------|
| 1 | direct-nara DB 스냅샷 + `DATABASES['direct_nara']` 설정 |
| 2 | Profile/Equipment 에 legacy 필드 있으면 유지, 없으면 복구 |
| 3 | `import_direct_nara` 로 회원·매물 이관 (비밀번호는 unusable 권장) |
| 4 | 이미지 이관 명령으로 사진까지 반영 |
| 5 | 이관 회원: 관리자 임시 비밀번호 또는 “전화번호+이메일 인증” 후 비밀번호 설정 |
| 6 | 로그인 후 “정식 회원 전환”에서 **새 아이디·이메일·비밀번호** 설정 → 일반 회원과 동일 이용 |

이렇게 하면 direct-nara 회원·매물을 현재 사이트 DB에 적용하고, 정식 회원 전환 절차로 일원화할 수 있습니다.
